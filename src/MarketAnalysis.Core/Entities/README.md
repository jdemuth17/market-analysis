# Entities — AiPrediction Schema & Lifecycle

## AiPrediction Entity

### Schema

```csharp
public class AiPrediction
{
    public long Id { get; set; }
    public int StockId { get; set; }
    public Stock Stock { get; set; } = null!;

    // Prediction metadata
    public DateOnly PredictionDate { get; set; }
    public string ModelUsed { get; set; } = "";          // e.g., "deepseek-v3.2"
    public bool IsAutoGenerated { get; set; }            // true = scan, false = manual
    public DateTime CreatedAtUtc { get; set; }

    // LLM-generated analysis
    public string Summary { get; set; } = "";            // 2-3 sentence overview
    public string Outlook { get; set; } = "";            // Bullish/Bearish/Neutral
    public string Recommendation { get; set; } = "";     // Buy/Hold/Sell
    public double Confidence { get; set; }               // 0.0-1.0

    public string KeyFactorsJson { get; set; } = "[]";   // List<string> as JSON
    public string RiskFactorsJson { get; set; } = "[]";  // List<string> as JSON

    // LLM-suggested trade levels
    public decimal EntryPrice { get; set; }
    public decimal StopLoss { get; set; }
    public decimal ProfitTarget { get; set; }
    public decimal ExitPrice { get; set; }
    public string TradeRationale { get; set; } = "";

    public string PredictedDirection { get; set; } = ""; // "bullish" or "bearish"

    // Backtesting outcomes (set by evaluation job)
    public decimal? ActualPriceAt5Days { get; set; }
    public decimal? ActualPriceAt10Days { get; set; }
    public decimal? ActualPriceAt30Days { get; set; }

    public string? OutcomeAt5Days { get; set; }          // "hit", "miss", "neutral"
    public string? OutcomeAt10Days { get; set; }
    public string? OutcomeAt30Days { get; set; }

    public DateTime? EvaluatedAt { get; set; }
}
```

### Lifecycle

#### 1. Creation

**Manual (User-Initiated)**:
- User clicks "Generate AI Report" in StockDetail view
- .NET calls Python service `/ai-analysis` endpoint
- Python queries Ollama (deepseek-v3.2) with price/technical/fundamental/sentiment context
- Response saved to `AiPrediction` table with `IsAutoGenerated = false`

**Auto-Generated (Daily Scan)**:
- Hangfire job runs daily scan at 6 PM ET
- Top 5 stocks per category (Growth, Momentum, Value, Reversal) selected
- Python service generates 20 reports total
- Saved with `IsAutoGenerated = true`

**Fields Set at Creation**:
- `PredictionDate`: Date prediction was made
- `EntryPrice`: Current stock price at time of prediction
- `ModelUsed`: "deepseek-v3.2"
- `PredictedDirection`: "bullish" or "bearish" based on LLM outlook
- `Summary`, `Outlook`, `KeyFactorsJson`, `RiskFactorsJson`, `Recommendation`, `Confidence`, `TradeRationale`: All populated from LLM structured output

#### 2. Evaluation (Backtesting)

**Scheduled Job**: Hangfire recurring job `"daily-prediction-eval"` runs at **7 PM ET daily** (`"0 19 * * *"`).

**Process** (in `PredictionEvaluationService.EvaluateAsync()`):
1. Query unevaluated predictions where `PredictionDate + N days <= today` for N ∈ {5, 10, 30}
2. Fetch actual price at evaluation date (± 5 days to handle weekends/holidays)
3. Calculate price change: `(actualPrice - entryPrice) / entryPrice`
4. Apply neutral threshold: `Math.Abs(changePct) < 0.005` → outcome = "neutral"
   - **Why 0.5%**: Daily noise typically ±0.3%. Threshold above noise floor distinguishes flat from directional movement. Aligns with trading friction costs.
5. For non-neutral outcomes:
   - Actual direction = `changePct > 0 ? "bullish" : "bearish"`
   - Outcome = actual == predicted ? "hit" : "miss"
6. Update corresponding `ActualPriceAtNDays`, `OutcomeAtNDays` fields
7. Set `EvaluatedAt = DateTime.UtcNow`

**Isolation**: Evaluation runs in transaction with `ReadCommitted` to prevent concurrent update race.

#### 3. Accuracy Tracking

**Aggregation**: `GetAccuracyStatsAsync()` computes hit/miss/neutral counts across all evaluated predictions.

**UI Display**: StockDetail and Predictions views show accuracy percentages at 5/10/30 day horizons.

## Relationship to Stock Entity

`AiPrediction` has foreign key to `Stock` (many-to-one):
- `StockId` references `Stock.Id`
- Navigation property `Stock` for EF Core queries
- `Stock.AiPredictions` collection (inverse navigation, not explicitly defined in Stock entity)

**Cascade Behavior**: If `Stock` deleted, associated predictions are **not** cascade-deleted. Predictions persist for historical backtesting data. Stock entity only soft-deletes via `IsActive = false`.

## Invariants

- `OutcomeAtNDays` fields are immutable after first evaluation (set once, never updated).
- `EntryPrice` must match stock price at `PredictionDate` (verified on creation).
- `StopLoss < EntryPrice < ProfitTarget` (validated by `TradeLevelResponse.model_validator` in Python).
- `PredictedDirection` must be "bullish" or "bearish" (lowercase, exact match required for evaluation logic).
